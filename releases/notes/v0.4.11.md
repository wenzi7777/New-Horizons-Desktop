# New Horizons Desktop v0.4.11

## Fixes

### Faster offline detection (desktop side)

- Follow-up to the gateway fix (New Horizons Gateway v0.2.8). The true root cause
  of devices staying "normal / last seen = now" after power-off was the gateway
  holding `connected: true` for 90s; that is fixed in the gateway.
- On the desktop side, tightened the now-redundant stacked timers so the offline
  state surfaces promptly once the gateway reports the device gone:
  - `FINDME_DISCONNECT_GRACE_SEC` 15s → 5s (only rides out a single transient
    disconnected snapshot; the gateway already debounces upstream).
  - `STALE_DEVICE_TIMEOUT_SEC` 30s → 12s and `STALE_DEVICE_CHECK_INTERVAL_SEC`
    15s → 5s (watchdog clears `gateway_connected` faster once `last_live_seen_at`
    goes stale).
- Net effect: a powered-off gateway-relayed device now shows offline in ~20s
  end-to-end (gateway ~16s + desktop tail), and direct-UDP devices age out faster
  too.

## Verification

- Backend unit suite passes (gateway disconnect/stale/grace regressions all
  green under the tightened constants).
- Real-time end-to-end check (gateway state → desktop service) confirms a
  powered-off device flips to disconnected within the window.

## Deployment

- **Requires New Horizons Gateway v0.2.8+** for the full fix. The desktop change
  alone does not resolve the 90s delay — the gateway host must also be updated
  and restarted.
