# New Horizons Desktop v0.3.4

## Changes

### Calibration: Switch level unit from IMADA N to MPX5050 kPa

The calibration level (the value stored per pressure point in the firmware) now uses the **MPX5050 gas pressure reading (kPa)** instead of the IMADA force gauge reading (N).

**Why:**
The previous approach used IMADA N as the calibration reference, which maps every sensor to "what force did the IMADA probe measure at this pressure." However, the IMADA probe area is much larger than a single sensor cell, so the N value is not the actual force on each cell — it is a force reading tied to the probe geometry. This made calibrated sensor outputs difficult to interpret as true pressure.

kPa is a scale-independent surface pressure unit; using it as the calibration reference makes the output directly represent the applied surface pressure, regardless of probe or sensor cell area.

**What changed:**
- `runCalibration()`: level is now `settledKpa` (MPX5050 actual reading) instead of `referenceN` (IMADA reading).
- Level browser now shows "X kPa" labels.
- Manual workbench level input now expects kPa values.

**Note:** Any existing calibration profiles were built with N-based levels. Clear the profile and re-calibrate to get kPa-based levels.
