# New Horizons Desktop v0.4.13

## Changes

### Fix "Save raw reading setting" reporting `unknown_command`

- The Device Settings "Raw reading stream" panel could not save: pressing
  **Save raw reading setting** returned an `unknown_command` error even though
  the firmware fully supports the command.
- Root cause: the backend has two command allowlists. The `set_raw_adc`
  command was present in `terminal.py`'s `DEVICE_COMMAND_ALLOWLIST` but missing
  from the mode-aware gate in `service.py` (`NORMAL_COMMANDS`), so it was
  rejected before reaching the device.
- Added `set_raw_adc` to `NORMAL_COMMANDS` (it is a plain config command that
  works in normal mode, like `set_filter`/`set_imu`), and added a regression
  test covering it.

### External LED preset adaptation

- New external-LED presets and a `solid_marker` colour picker in the device
  settings UI, with matching backend/terminal handling and documentation.

### Simplified Chinese wiki

- Added Simplified Chinese (`zh-CN`) wiki translations for all three devices.

  `test_set_raw_adc_allowed_in_normal_mode`).
- `python3 -m unittest tests.test_terminal -q` — passes.
