# New Horizons Desktop v0.3.5

## Changes

### Calibration: Use differential kPa as level (baseline-subtracted)

The calibration level is now stored as **differential kPa** (pressure above the
tare baseline) rather than absolute kPa.

After tare capture, the current MPX5050 reading is recorded as `baselineKpa`.
Each subsequent pressure point stores `level = settledKpa - baselineKpa`.

This ensures the PCHIP curve origin (ADC_diff=0 → level=0) is physically
consistent: output 0 means "at baseline pressure", and e.g. output 6.5 means
"6.5 kPa above baseline". This matches the behaviour of the original IMADA-N
calibration where the force gauge read 0 N at baseline.

The calibration log now shows both absolute and differential kPa per point.
