# New Horizons Desktop v0.4.5

## Fixes

### Device reconnect state and Launchpad grouping

- Split device `last_seen_at` from `last_live_seen_at` so disconnect and
  transport bookkeeping no longer make offline devices look like they were
  just seen.
- Added explicit `reconnecting` device state handling in the Desktop UI,
  including a 15-second reconnect grace window before devices fall back to
  `offline`.
- Kept the real device mode (`normal`, `maintenance`, `safe_maintenance`)
  separate from connection-state rendering so unavailable control paths no
  longer show as `normal`.
- Added persistent per-device custom grouping with a new Desktop API and
  Launchpad folder view.
- Added default i18n-backed `Online` and `Offline` Launchpad folders, while
  keeping user-created group names as raw strings.
- Added device group controls in Settings and disabled live query / control
  actions during both reconnecting and offline states.

## Verification

- Verified backend gateway regression coverage for disconnect/live-seen
  semantics and device-group persistence.
- Verified frontend static coverage for Launchpad grouping, Settings group
  controls, reconnecting state handling, and live-query guard behavior.
- Verified the frontend production build.
