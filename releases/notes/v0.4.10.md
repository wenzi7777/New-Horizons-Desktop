# New Horizons Desktop v0.4.10

## Fixes

### Offline detection for powered-off devices

- Fixed devices staying shown as `normal` with `last seen = now` after being
  powered off (both soft shutdown and a hard power cut) when relayed through a
  gateway.
- Root cause: a gateway keeps re-mentioning a disconnected device in every ~5s
  snapshot until its own ~90s stale window, which kept refreshing `last_seen_at`.
  Any freshness check keyed off `last_seen_at` therefore never expired, so the
  findme reconnect grace — on both the backend and the frontend — never elapsed
  and the device looked online forever.
- Backend (`service.py`): the findme disconnect grace and the stale-device
  watchdog now measure age from `last_live_seen_at` (only advanced while the
  device is *genuinely* connected), and the grace no longer advances that clock
  itself. Once a device truly drops, `gateway_connected` flips to `False` and
  `last_live_seen_at` freezes.
- Frontend (`device.ts` / new `connectionState.ts`): the `reconnecting`/`offline`
  decision now keys off the genuinely-online clock (`last_live_seen_at`) instead
  of `last_seen_at`, so the UI leaves the amber "reconnecting" state and shows
  "offline" once the grace expires, with "last seen" frozen at the real time.
