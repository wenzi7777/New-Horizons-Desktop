# New Horizons Desktop v0.4.1

## Fixes

### Device Wiki content in deployed builds

- Included the repository `wiki/` tree in the Docker image so the backend
  default `/app/wiki` root contains device documentation after deployment.
- Fixed the Wiki preview panel so it shows an empty-state message instead of
  staying on `Loading` when no device or markdown document is available.
- Limited auto-selection to markdown files, avoiding accidental document loads
  against directory entries.

## Verification

- Added regression coverage that the Docker image copies `wiki/`.
- Added frontend static coverage for the Wiki preview empty/loading states.
