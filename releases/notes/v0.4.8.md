# New Horizons Desktop v0.4.8

## Features

### Raw reading (rawADC) support

- Parsed the new firmware packet flag `0x08`: when a device streams raw
  readings, `packet_parser` now returns a `raw_adc` array alongside the
  calibrated `p` (level) values, inferring the doubled matrix layout from the
  flag.
- Surfaced raw readings end to end: the visualization stream carries `raw_adc`,
  CSV exports gain `RawADC_*` columns when present, and the 2D heatmap shows the
  raw value next to the calibrated level for side-by-side comparison.
- Added a Device Settings toggle (`set_raw_adc` control command) to enable or
  disable the raw stream per device. **Off by default**; the desktop reflects
  the persisted `stream_raw_adc` state reported by the firmware.
- Added English and Japanese strings for the new setting.

## Verification

- Verified the backend unit test suite, including new raw-packet parsing
  coverage in `tests/test_arduino_packet_v3.py`.
- Verified the frontend production build (TypeScript check + Vite bundle).

## Compatibility

- Requires New Horizons OS firmware v0.10.0+ for the raw stream. Devices on
  older firmware are unaffected; the toggle stays off and packets parse exactly
  as before.
