# New Horizons Desktop v0.4.4

## Fixes

### CSV recording timestamps

- Changed recorded CSV files to use `timestamp_ms` as the first column header.
- The first CSV column now stores Unix epoch milliseconds from Desktop
  receive time or `received_at`, instead of firmware packet uptime.
- Preserved pressure and IMU column ordering so existing analysis scripts do
  not need to move sensor fields.

## Verification

- Added backend regression coverage for CSV header and epoch-ms timestamp
  output.
- Verified backend recording tests for full-frame CSV writes.
- Verified the frontend test script and production build.
