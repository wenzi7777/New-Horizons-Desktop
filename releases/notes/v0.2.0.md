# New Horizons Desktop v0.2.0

## Changes

### Pressure Calibration: Automated Multi-Point Workflow

- Added `PressureCalibrationPage` with a frontend state machine that drives
  automated multi-point pressure calibration against a remote pneumatic lab
  server (Raspberry Pi API).
- Added backend proxy routes `/api/pressure-cal/*` and `pressure_cal_client.py`
  (urllib-based, 45 kPa safety limit hard-coded).
- Added `pressure_cal_settings.py` for persistent URL and token storage.

### Pressure Calibration: Integrated into Maintenance Section

- Moved the pressure calibration workflow from a standalone Launchpad page into
  the device Settings → Maintenance tab, alongside the existing calibration
  workbench. Styled to match `settings-card` / `settings-detail-header` /
  `metric-row` patterns.

### Pressure Calibration: Built-in Server Presets

- Replaced the manual URL / token input fields with a server preset selector.
  Built-in presets (Lab Pi Server) store credentials in the backend and never
  expose them to the JS bundle. Users can also select "Custom" to enter their
  own URL and token.
- Old `{url, token}` settings are auto-migrated to `{preset: "custom", url, token}`.
